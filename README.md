# 手机握持方向识别算法（左右手判断）
基于手机陀螺仪（Gyroscope）和加速度传感器（Accelerometer）数据，实现手机左右手握持状态的自动判断，配套演示客户端可直观验证算法效果。

## 项目概述
本项目核心目标是通过传感器原生数据，无额外硬件依赖地判断用户当前是左手还是右手握持手机。
> ⚠️ 现阶段说明：算法的判断成功率、响应速度仍有较大优化空间，当前版本仅作技术验证与演示。

## 核心功能
- 双算法协同判断左右手握持状态；
- 配套客户端可视化算法执行过程、传感器数据变化；
- 模拟来电悬浮窗功能，还原算法在真实场景（如华为智感握持）的应用效果。

## 技术实现
### 1. 算法模块（核心）
项目包含两个独立算法，组合使用：

| 算法类型       | 依赖传感器     | 核心原理                                                                 | 特点                               |
|----------------|----------------|--------------------------------------------------------------------------|------------------------------------|
| 主算法（算法 1）| 陀螺仪&加速器传感器| 采集手机姿态稳定期的 Roll 角作为基准值；之后通过 Roll 值的偏移方向判断左右手握持。<br>基准值实时计算,并采取滤波处理。<br>加速度在一定数值范围内才开始重新判断，增加正确率。<br>基准值和加速度判断异步刷新，基准值1.2s刷新一次，加速度0.7s判断一次 | 逻辑相对精准，但受手机姿态稳定性影响较大 |
| 辅算法（算法 2）| 加速度传感器   | 识别手机横向移动方向，通过移动轨迹特征简易判断握持手。                     | 响应快，精度较低，作为主算法补充 |

### 2. 客户端模块（演示）
- **核心作用**：可视化传感器数据、算法判断流程及结果，辅助开发者调试优化；
- **特色功能**：模拟来电悬浮窗（需手动开启悬浮窗权限），还原算法在实际场景的应用；
- **开发说明**：客户端代码大量借助 AI 辅助开发，仅作为算法演示 Demo，未做生产级优化。

## 运行说明
1. 编译运行客户端需 Android Studio 环境；
2. 运行前需授予「传感器权限」（默认开启）「悬浮窗权限」（需手动授权）；
3. 建议在稳定场景下测试（如手持静止状态），减少环境干扰。

## 待优化方向
- 提升算法判断成功率，这是重中之重；
- 优化算法响应速度，降低判断延迟；
- 更好的融合算法结果，比如增加动态权重判断，提升整体判断精度；
- 可以考虑增加更多独立判断算法，但是尽可能只使用通用传感器；
- 让客户端更美观。
